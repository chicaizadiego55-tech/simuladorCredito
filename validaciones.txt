SPROMPT 1 — Validaciones básicas por campo (onblur)
----------------------------------------------------------
Tengo un formulario HTML de simulador de crédito con los siguientes ids de inputs:
txtIngresos, txtArriendo, txtAlimentacion, txtVarios, txtMonto, txtPlazo, txtTasaInteres.
 
Quiero validar cada input con las siguientes reglas generales:
- No puede estar vacío
- Solo se permiten números (enteros o decimales positivos)
- Máximo 5 dígitos en la parte entera
 
Condiciones técnicas:
- No usar alert(), mostrar el mensaje debajo de cada input
- El mensaje de error debe aparecer en color rojo y en cursiva
- Usar los ids existentes sin modificarlos
- Usar el evento onblur para validar cuando el usuario sale del input
- Crear dinámicamente un span de error debajo de cada campo

PROMPT 3 — Ejecutar validaciones al presionar el botón Calcular
----------------------------------------------------------
Modifica el comportamiento del botón con id btnCalcularCredito para que:
1. Primero ejecute todas las validaciones del formulario
2. Solo si todas las validaciones pasan sin errores, ejecute la función calcular()
3. Si hay errores, NO ejecute calcular() y muestre todos los mensajes de error
   debajo de cada campo correspondiente al mismo tiempo
4. Limpiar todos los mensajes de error al inicio de cada validación nueva
5. No usar alert() en ningún momento
6. Los mensajes deben aparecer en rojo e itálica debajo de cada input
